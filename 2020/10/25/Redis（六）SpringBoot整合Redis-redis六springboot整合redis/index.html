Pom分析

spring-boot-starter-data-redis点进去时候发现，没有jedis的依赖

SpringBoot2.X之后，原来使用的jedis被替换成了lettuce

Jedis：采用的直连，多个线程操作的话。是不安全的，如果想要避免不安全的情况，可以使用Jedis Pool，但这是（BIO）阻塞IO

Lettuce：采用Netty，实例可以再多个线程中进行共享，不存在线程不安全的情况！可以线程数量，更像NIO模式
